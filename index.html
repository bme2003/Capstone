About

ReLiANT is a student engineering team focused on specification-driven testing. Our goal is to turn formal TLA+ models into concrete, replayable NVMe test sequences. We emphasize clarity, determinism, and coverage-so findings are repeatable and actionable.

  • TLC “generate” mode to handle non-determinism in specs
  • Rust executor to drive nvme commands against SSD DUTs
  • Instrumentation to map spec states between device/system states
  • Stretch: automatic re-test for failing sequences

Project

Objectives

  1. Model target NVMe behaviors in TLA+ (PlusCal as needed).
  2. Use TLC to generate valid execution traces.
  3. Translate traces into Rust calls to nvme-cli.
  4. Record results with sufficient metadata for replay/debug.

Milestones

  • M1: Minimal TLA+ model + TLC runbook
  • M2: Rust executor + local DUT smoke tests
  • M3: Instrumentation & reporting
  • M4: Expanded coverage + stretch goal
